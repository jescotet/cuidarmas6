Propuesta de Desarrollo para Cuidar+

Una solución innovadora para conectar generaciones, aliviar la soledad y ofrecer tranquilidad a las familias.

El Proyecto: Visión y Objetivo

Esta sección detalla el propósito fundamental de Cuidar+, identificando el problema que resuelve y las personas a las que se dirige.

Objetivo Principal

Diseñar un asistente conversacional por voz en WhatsApp para personas mayores, con un Portal Web para que sus cuidadores puedan personalizar la experiencia y monitorizar las interacciones.

Perfiles de Usuario

  • Primario: El familiar o cuidador que usa el Portal Web.
  • Secundario: La persona mayor que usa WhatsApp.

Customer Journey

  1. 1. Registro

    El cuidador se registra en la WebApp.

  2. 2. Creación de Perfil

    Crea un perfil para la persona mayor.

  3. 3. Personalización

    Define el tono y temas de conversación.

  4. 4. Activación

    Valida el WhatsApp y el asistente se activa.

  5. 5. Monitorización

    El cuidador supervisa la actividad y alertas.

Solución Técnica

Aquí exploramos el "cómo": la arquitectura y tecnologías seleccionadas para construir Cuidar+.

Diagrama de Arquitectura

Usuario: Persona Mayor

Interactúa únicamente por audios de WhatsApp.

API de WhatsApp

(Gupshup)

Recibe audios y envía respuestas. Conecta con el cuidador.

WebApp Cuidador

(React)

Portal para configurar y monitorizar.

Speech-to-Text

(Whisper API)

Convierte el audio del usuario a texto.

Backend DB

(PostgreSQL/Firestore)

Almacena logs, perfiles, y alertas.

LLM: GPT-4o API

(OpenAI)

Genera la respuesta conversacional.

Text-to-Speech

(Play.ht/Google)

Convierte el texto de la IA a un audio natural.

Stack Tecnológico

WebApp (Frontend)

React (con Vite)

Ideal para crear interfaces interactivas y dinámicas.

Backend (Middleware)

Node.js (con Express)

Perfecto para orquestar llamadas a APIs de forma asíncrona.

Base de Datos

Firestore (Firebase)

NoSQL, escalable y en tiempo real para monitorización en vivo.

Speech-to-Text

Whisper API (OpenAI)

Transcripciones de alta precisión para un habla natural.

Modelo de Lenguaje (LLM)

GPT-4o (OpenAI)

Capacidad multimodal y de conversación natural.

Text-to-Speech

ElevenLabs / PlayHT

Voces hiperrealistas, cálidas y con entonación natural.

Hoja de Ruta del MVP

                
Presentamos un plan de desarrollo de 10-12 semanas, desglosado en fases claras con hitos específicos.

            

            
1
Semanas 1-3: Setup y Backend Core
Configuración de APIs (WhatsApp) y desarrollo del middleware en Node.js.
2
Semanas 4-5: Integración IA
Integración de las APIs de STT, LLM y TTS. Creación del flujo completo de audio.
3
Semanas 6-9: Desarrollo WebApp
Creación de todas las pantallas en React y conexión a la base de datos.
4
Semanas 10-12: Pruebas y Ajustes
QA testing exhaustivo y ajustes finales basados en el feedback.
